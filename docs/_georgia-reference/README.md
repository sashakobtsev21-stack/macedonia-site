# `_georgia-reference/` — архив Georgia Guidebook (только как шаблоны)

Эти файлы **не относятся к проекту North Macedonia Guidebook напрямую**. Это
документы исходного проекта **Georgia Guidebook** (`gruzia-site`), на движке
которого форкнут этот репозиторий. Они сохранены как **референс и шаблоны**:
структура контент-плана, формат брифов, методология аудита/роадмапа, проектная
память — всё это переиспользуется при моделировании контента под Северную
Македонию (следующий шаг), но **содержат грузинские факты/темы и не являются
актуальной правдой по этому сайту**.

Актуальные системные документы движка живут в корне репозитория: `SPEC.md`,
`CLAUDE.md`, `CONTENT_GUIDE.md`, `SETUP-GUIDE.md`.

## Что здесь лежит

| Файл / папка | Что это (для Грузии) |
|---|---|
| `AUDIT.md` | Сводный аудит сайта (журнал заходов) |
| `PROGRESS.md` | Статус и история разработки |
| `HANDOFF.md` | Снимок состояния для продолжения в новом чате |
| `ROADMAP.md`, `ROADMAP-GROWTH-2026.md` | Планы доработок и роста |
| `TODO-affiliates.md` | Бэклог партнёрской монетизации |
| `INSTRUKCIYA-KONTENT.md` | Инструкция по контенту для владельца |
| `KALENDAR.md` | Календарь работ по дням |
| `CONTENT_PLAN.md` | План тем контента |
| `GROWTH-MONETIZATION-ANALYSIS-2026-06-20.md` | Анализ роста и монетизации |
| `briefs/` | Брифы статей (вход контент-конвейера) |
| `memory/` | Зеркало проектной памяти Georgia Guidebook |

> При перемоделировании под Македонию опирайся на **форму** этих документов, а не
> на их грузинское содержимое. Новую правду писать в свежие документы в корне и в
> `docs/memory/`.
